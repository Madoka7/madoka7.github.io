2017-05-22 • Categories: Experience

Talk is cheap, come with CODE
废话少说，放码过来

来自雍高鹏的编程技巧经验课程
1、问题驱动的学习，在解决问题过程中提升能力，再次遇到问题之后就能快速解决新问题。
2、Level one? 就是说快速解决问题的能力的获取是最简单的初级问题。
来自长者的建议：

  • Do not be afraid
  • Be lazy (let computer work more)
  • Be the leader

3、两个段子：包子馅的包子；斐波那契汤
4、更高级的能力：除了实现，更多考虑算法的选取，考虑数据结构的问题。

Bad programmers worry about the Code. Good programmers worry about data structures and their relationships.
三流程序员只关心代码的实现，好的程序员则关心数据结构及它们间的关系。

他现在说的东西我已经听不懂了。。。
什么Competition coefficient
什么References dupliation remove
什么…没记下来
5、列举了一些面试中可能会遇到的东西，如果都会了就可以云云

其间还错过了了什么高级图之类的东西
人生目标和你中间会有一个大坑，需要用ACM之类的东西填满。
6、（终于有一个能听懂的）先完成后完美
7、编程写好了之后有什么帮助：

  • 可以锻炼许多意志品质
  • 没记下来
  • 有朝一日让所有人生活更美好

正文结束

王老师说：只有把自己推上去。

一个化工系的老师，以前在大家的队里，打不上比赛。后来退出了，进入了化工队，结果现在是德云一哥。这就是把自己推上去的帮助；前提是你有变强的需求。
